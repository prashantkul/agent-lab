Reviewer Help Guide

Everything you need to know about reviewing course materials

Quick Start

  1. 1 Select a module from the available list
  2. 2 Review the PDF course material
  3. 3 Complete the coding assignment via GitHub Classroom
  4. 4 Submit your feedback on the module

As a reviewer, you can select up to 2 modules to review at a time.

Modules in Pilot Review status are available for selection.

To add a second module, click the "Add 2nd Module" button in the header.

You can switch between your selected modules using the module switcher at the top of the dashboard.

Step 1: Click the Accept Assignment button to open GitHub Classroom.

Step 2: Accept the assignment - this creates your personal repository.

Step 3: Clone your repository to your local machine:

git clone https://github.com/your-org/assignment-your-username.git

Step 4: Complete the assignment following the instructions in the README.

Step 5: Push your changes to trigger automatic grading:

git add .
git commit -m "Complete assignment"
git push

Step 6: View your grade in the GitHub Actions tab of your repository.

Your feedback helps improve the course materials for future students.

What to include:

  • Link to your completed GitHub repository
  • Clarity rating (1-5) - How clear were the instructions?
  • Difficulty rating (1-5) - How challenging was the material?
  • Time spent on the assignment
  • Detailed comments on what worked well and what could be improved

You can update your feedback anytime before releasing the module.

When you're done reviewing a module, click "Release" to free up your slot.

Before releasing:

  • Make sure you've submitted your feedback
  • Ensure your GitHub assignment is complete

Note: Releasing a module is permanent. You won't be able to re-select it.

Still have questions?

Contact the course instructor for help.
